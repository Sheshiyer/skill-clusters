# ExtractFromSource

Use this workflow when a reel, demo, or creator video contains a tool stack that should become a reusable workflow skill.

## Steps

### 1. Strip the source down to operational claims

Ignore:
- promotional CTA
- affiliate hooks
- vanity framing

Keep:
- explicit tool names
- sequence of use
- on-screen UI evidence
- repeated outputs the workflow is supposed to generate

### 2. Separate the workflow layers

Classify each tool into one of these buckets:
- research or demand discovery
- audience language discovery
- browser capture or gated UI access
- orchestration or downstream normalization
- optional generation layer

### 3. Rebuild the workflow in neutral language

Translate creator phrasing into a reusable pattern:
- input
- step order
- outputs
- automation handoff
- failure checkpoints
- user-direction branches

### 4. Write the reusable skill

Capture:
- when to use it
- the concrete sequence
- deliverables
- failure modes
- source trace

### 5. Verify against extracted evidence

Before finalizing:
- confirm the caption still supports the tool list
- confirm OCR supports at least part of the on-screen flow
- remove claims that exist only in your inference
- if replacing a creator tool with an agent-executable implementation, document the substitution explicitly
